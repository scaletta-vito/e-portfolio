Unit 5: Data Integration

This unit covered techniques for combining data from multiple sources, addressing challenges of heterogeneity, schema conflicts, and entity resolution. We explored ETL (Extract, Transform, Load) processes and data pipeline design.

Learning Outcomes

  • Design ETL processes for data integration
  • Handle schema heterogeneity across sources
  • Apply entity resolution techniques
  • Build reproducible data pipelines

Unit Reflection

Criteria Experience Learning
What?
(Description)		 In this unit, I studied data integration concepts including ETL workflows, schema mapping, and handling data from heterogeneous sources. The readings covered both theoretical frameworks and practical implementation approaches. The unit explained why integration is challenging: different sources use different identifiers, formats, and semantic conventions. Understanding these challenges informed the design of our team project's data pipeline.
So what?
(Analysis)		 Attempting to merge datasets in our team project revealed integration challenges the readings described. Matching records across sources without unique identifiers required developing fuzzy matching rules, which was more complex than anticipated. I learned that successful integration requires upfront planning and clear documentation of transformation logic. The concept of "data lineage" became importantâ€”knowing where each piece of data originated and how it was transformed.
Now what?
(Outcome)		 I will invest more time in source profiling before beginning integration work. Creating detailed mapping documents will become standard practice to ensure transformations are traceable and reversible. Professionally, I plan to explore dedicated ETL tools and frameworks that handle common integration patterns, rather than building custom solutions from scratch each time.

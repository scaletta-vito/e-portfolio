Unit 6: Project Proposal

This unit marked the beginning of the collaborative project phase. Teams were formed, project domains selected, and initial proposals developed. This was a critical milestone that established the foundation for the final project deliverable in Unit 11.

Key Deliverable: Team project proposal outlining scope, data sources, methodology, and expected outcomes. See Project Evaluation for comparison with final delivery.

Learning Outcomes

  • Form effective teams for collaborative data projects
  • Define realistic project scope based on available resources
  • Develop clear project proposals with measurable objectives
  • Apply database design principles to real-world scenarios

Unit Reflection

Criteria Experience Learning
What?
(Description)		 In this unit, I explored the principles of designing relational databases, focusing on key fields, data integrity, and normalization techniques. I learned how to identify anomalies in databases and practiced creating logical and physical database designs for our team proposal. The unit introduced key database concepts such as relational structures, normalization forms, and the importance of eliminating data anomalies. I gained a clearer understanding of the terminology and processes involved in constructing robust database architectures.
So what?
(Analysis)		 Working through database design exercises made me realize how crucial normalization is for maintaining data integrity and preventing redundancy. Understanding key fields helped me see how tables relate and how data retrieval depends on well-structured relationships. This deepened my knowledge of database theory, highlighting the practical impact of normalization on real-world data storage and retrieval. It also emphasized the need for careful planning in both logical and physical design stages to ensure efficient and scalable database systems.
Now what?
(Outcome)		 In future projects, I plan to apply normalization principles more rigorously and double-check for anomalies during design. I want to practice creating both logical and physical database schemas to enhance my ability to design effective data architectures. Professionally, I aim to further study advanced database design techniques, including optimization and indexing strategies, to improve performance. I also want to explore tools that assist in database modeling and normalization to streamline the design process.

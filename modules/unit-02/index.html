Unit 2: Data Collection

This unit focused on data collection methodologies, exploring various data sources, formats, and extraction techniques. We examined how organisations gather data from structured and unstructured sources, and the challenges associated with collecting data at scale.

Learning Outcomes

  • Identify different types of data sources (internal, external, primary, secondary)
  • Understand data formats including CSV, JSON, XML, and databases
  • Apply data extraction techniques using Python
  • Recognise quality considerations during data collection

Unit Reflection

Criteria Experience Learning
What?
(Description)		 This unit covered data collection methods and the variety of sources from which data can be extracted. I worked through exercises on reading data from different file formats using Python libraries. The unit explained how data collection forms the foundation of any analytics project. Understanding source diversity and format differences is essential for planning extraction strategies effectively.
So what?
(Analysis)		 Working with multiple data formats revealed how format inconsistencies create immediate challenges. Now what?
(Outcome)		 I plan to build a personal library of extraction scripts for common formats that I can reuse across projects. Documenting source characteristics upfront will become part of my standard practice. Professionally, I want to establish data collection standards for my organisation that ensure quality from the start. This includes validation checks at the point of collection rather than waiting for analysis stages.

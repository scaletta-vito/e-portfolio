Unit 3: Data Storage

This unit examined data storage architectures, comparing relational databases, NoSQL solutions, and distributed storage systems. We explored criteria for selecting appropriate storage solutions based on data characteristics and use case requirements.

Learning Outcomes

  • Compare relational and non-relational database systems
  • Understand structured vs unstructured data storage requirements
  • Evaluate storage solutions against performance and scalability criteria
  • Apply Pandas for data manipulation and storage operations

Unit Reflection

Criteria Experience Learning
What?
(Description)		 In this unit, I learned about different data storage systems and how to choose between them based on data characteristics. I completed practical exercises using Pandas to read, manipulate, and store data in various formats. The unit covered structured, semi-structured, and unstructured data categories, explaining why each requires different storage approaches. Key concepts included data lakes, data warehouses, and the trade-offs between them.
So what?
(Analysis)		 The hands-on work with Pandas demonstrated how data structures affect query performance. I experienced firsthand how poorly chosen data types and storage formats can slow down operations significantly. Understanding the distinction between OLTP and OLAP systems clarified why organisations often maintain multiple storage solutions. The concept of "schema-on-read" versus "schema-on-write" helped explain different design philosophies.
Now what?
(Outcome)		 I will practice more with Pandas operations to improve my data manipulation efficiency. Understanding indexing and data types better will help me make smarter storage decisions. For future projects, I will conduct storage requirement analysis before implementation rather than defaulting to familiar solutions. This includes considering scale, query patterns, and consistency requirements.

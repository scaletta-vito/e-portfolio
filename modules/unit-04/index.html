Unit 4: Data Cleaning

This unit focused on data quality issues and cleaning techniques. We examined common problems including missing values, duplicates, inconsistencies, and outliers, along with systematic approaches to address them.

Learning Outcomes

  • Identify common data quality issues
  • Apply appropriate strategies for handling missing data
  • Detect and remove duplicate records
  • Standardise and validate data using Python

Unit Reflection

Criteria Experience Learning
What?
(Description)		 In this unit, I studied data cleaning techniques and completed the data management pipeline quiz. I practiced identifying and resolving data quality issues using Pandas functions like fillna(), drop_duplicates(), and data type conversions. The unit covered why data cleaning typically consumes 60-80% of data project time. Understanding the distinction between missing completely at random (MCAR), missing at random (MAR), and missing not at random (MNAR) informed appropriate handling strategies.
So what?
(Analysis)		 The quiz results highlighted gaps in my understanding of pipeline automation. Working through cleaning exercises showed how seemingly simple decisions (like choosing mean vs median imputation) significantly affect downstream analysis. I now appreciate that data cleaning requires domain knowledge, not just technical skills. Understanding why data is missing or inconsistent helps choose appropriate remediation strategies rather than applying generic fixes.
Now what?
(Outcome)		 I will develop a systematic checklist for data quality assessment that I apply consistently across projects. Documenting cleaning decisions and their rationale will become standard practice. Professionally, I plan to advocate for data quality investment early in project timelines rather than treating cleaning as a quick preliminary step.

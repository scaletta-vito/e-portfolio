Reflective Analysis

This reflection follows Rolfe et al.'s (2001) reflective framework, structured around three key questions: What? So what? Now what? This approach enables systematic analysis of experiences and their implications for future practice.

WHAT? — Description

The Deciphering Big Data module introduced me to the fundamental concepts, tools, and methodologies required for working with large-scale datasets. Over twelve units, I engaged with topics spanning the 4Vs of big data (Volume, Variety, Velocity, Veracity), data extraction from multiple sources, cleaning and transformation techniques using Python and Pandas, database design principles including normalisation to Third Normal Form, and compliance considerations under GDPR.

My primary technical work involved developing a data pipeline for the team project, which required extracting data from CSV and JSON sources, cleaning inconsistencies and handling missing values, designing a normalised relational database schema, and documenting the entire process for reproducibility. I also participated in collaborative discussions on data management challenges, GDPR compliance implications, and database design trade-offs, contributing posts and responding to peers' perspectives.

SO WHAT? — Analysis and Interpretation

Working through this module fundamentally changed my understanding of data as an organisational asset. Prior to this experience, I viewed data primarily through a storage lens—ensuring files were backed up and accessible. The module revealed that effective data management requires strategic thinking about quality, governance, and analytical readiness from the outset (Sardar and Pandey, 2024).

The team project proved particularly transformative. Initially, I underestimated the complexity of coordinating data work across multiple contributors. Our first attempts at collaborative cleaning resulted in version conflicts and inconsistent transformations. This experience aligned with Kolb's (1984) experiential learning cycle: the concrete experience of failure prompted reflective observation, leading to abstract conceptualisation of better practices, which we then tested through active experimentation with version control and documented procedures.

Emotionally, the module presented significant challenges. The Unit 6 proposal deadline coincided with professional responsibilities at the Innovation Centre, creating stress that affected my initial contributions. I felt frustration when my database design required substantial revision following peer feedback. However, Schön's (1983) concept of reflection-in-action helped me reframe these moments not as failures but as essential learning opportunities. The discomfort of receiving critical feedback ultimately strengthened both the project outcome and my capacity for professional growth.

Analysing my collaborative discussion contributions revealed an evolution in critical thinking. Early posts tended toward descriptive summaries of readings. By Unit 8's GDPR discussion, I was synthesising multiple sources, identifying tensions between compliance requirements and analytical utility, and proposing practical frameworks for resolution. This progression demonstrates development of higher-order analytical skills essential for data leadership roles.

NOW WHAT? — Future Actions and Development

This module has crystallised several priorities for my continued professional development. First, I recognise the need to deepen my technical proficiency with Python libraries beyond Pandas. The limitations I encountered when attempting more complex transformations highlighted gaps that additional study of NumPy and Scikit-learn would address (McKinney, 2022).

Second, the team collaboration challenges have motivated me to explore project management methodologies specifically designed for data work. I plan to investigate DataOps practices and tools that facilitate collaborative data pipelines with proper versioning and documentation. This knowledge will directly benefit my leadership role at the Innovation Centre, where I oversee teams working with governmental datasets.

Third, the GDPR discussions revealed the importance of maintaining current knowledge of regulatory frameworks. Data protection requirements continue to evolve, and my professional responsibilities require staying informed of changes that affect how we collect, process, and store citizen data. I have committed to completing a formal data protection certification within the next six months.

Finally, this reflective process itself has proven valuable. Following Rolfe et al.'s (2001) structured approach transformed what might have been superficial self-assessment into genuine insight. I intend to maintain a reflective journal for future professional development activities, using this framework to extract maximum learning from experiences both positive and challenging.

Impact on Professional Development

The skills and knowledge acquired through this module have immediate applicability to my role directing the Dost Digital Innovation Centre. We are currently planning a data modernisation initiative for government services, and the principles of database normalisation, data quality assurance, and GDPR compliance covered in this module directly inform that work. The experience of designing and evaluating a data pipeline provides a template I can adapt for organisational projects.

Perhaps more significantly, the module has shifted my perspective on data literacy as a leadership competency. I now recognise that effective digital transformation requires leaders who understand not just what data can do, but how it must be managed responsibly. This understanding will shape how I advocate for resources, structure teams, and evaluate proposals in my professional capacity.

Data Cleaning with Pandas

The following code demonstrates cleaning procedures applied to the citizen services dataset, addressing missing values, duplicates, and data type inconsistencies (McKinney, 2022).

import pandas as pd
import numpy as np

# Load raw data from CSV source
df = pd.read_csv('citizen_services.csv')

# Initial data profiling
print(f"Original shape: {df.shape}")
print(f"Missing values:\n{df.isnull().sum()}")

# Remove duplicate records based on citizen_id
df_clean = df.drop_duplicates(subset=['citizen_id'], keep='first')
print(f"Removed {len(df) - len(df_clean)} duplicate records")

# Handle missing values with appropriate strategies
# Numerical: median imputation for robustness against outliers
df_clean['service_rating'] = df_clean['service_rating'].fillna(
    df_clean['service_rating'].median()
)

# Categorical: mode imputation for most common value
df_clean['service_category'] = df_clean['service_category'].fillna(
    df_clean['service_category'].mode()[0]
)

# Standardise date format
df_clean['request_date'] = pd.to_datetime(
    df_clean['request_date'], 
    format='mixed',
    errors='coerce'
)

# Remove invalid dates
df_clean = df_clean.dropna(subset=['request_date'])

# Validate and clean text fields
df_clean['citizen_name'] = df_clean['citizen_name'].str.strip().str.title()

# Export cleaned dataset
df_clean.to_csv('citizen_services_cleaned.csv', index=False)
print(f"Final shape: {df_clean.shape}")

Automated Data Pipeline

The following script demonstrates an automated pipeline that extracts, transforms, and loads data while maintaining audit logs.

import pandas as pd
import json
from datetime import datetime

class DataPipeline:
    """ETL pipeline for citizen services data processing."""
    
    def __init__(self, config_path):
        with open(config_path) as f:
            self.config = json.load(f)
        self.audit_log = []
    
    def extract(self, source_path):
        """Extract data from CSV or JSON source."""
        if source_path.endswith('.csv'):
            data = pd.read_csv(source_path)
        elif source_path.endswith('.json'):
            data = pd.read_json(source_path)
        self._log(f"Extracted {len(data)} records from {source_path}")
        return data
    
    def transform(self, df):
        """Apply cleaning and transformation rules."""
        initial_count = len(df)
        
        # Apply configured transformations
        df = df.drop_duplicates()
        df = df.dropna(subset=self.config['required_fields'])
        
        self._log(f"Transformed: {initial_count} → {len(df)} records")
        return df
    
    def load(self, df, output_path):
        """Load processed data to destination."""
        df.to_csv(output_path, index=False)
        self._log(f"Loaded {len(df)} records to {output_path}")
    
    def _log(self, message):
        """Record audit log entry."""
        entry = {
            'timestamp': datetime.now().isoformat(),
            'message': message
        }
        self.audit_log.append(entry)
        print(f"[{entry['timestamp']}] {message}")

# Execute pipeline
pipeline = DataPipeline('pipeline_config.json')
raw_data = pipeline.extract('citizen_services.csv')
clean_data = pipeline.transform(raw_data)
pipeline.load(clean_data, 'processed_services.csv')
